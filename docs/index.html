    2.1The try Statement

    Syntax

    TryStatement[Yield, Await, Return]:try([lookahead ∉ { let [ }]Expression[+In, ?Yield, ?Await])Block[?Yield, ?Await, ?Return]Catch[?Yield, ?Await, ?Return]optFinally[?Yield, ?Await, ?Return]opt try(varVariableDeclarationList[+In, ?Yield, ?Await])Block[?Yield, ?Await, ?Return]Catch[?Yield, ?Await, ?Return]optFinally[?Yield, ?Await, ?Return]opt try(LetOrConstBindingList[+In, ?Yield, ?Await])Block[?Yield, ?Await, ?Return]Catch[?Yield, ?Await, ?Return]optFinally[?Yield, ?Await, ?Return]opt tryBlock[?Yield, ?Await, ?Return]Catch[?Yield, ?Await, ?Return] tryBlock[?Yield, ?Await, ?Return]Finally[?Yield, ?Await, ?Return] tryBlock[?Yield, ?Await, ?Return]Catch[?Yield, ?Await, ?Return]Finally[?Yield, ?Await, ?Return]

    2.2Static Semantics: Early Errors

    TryStatement:try(LetOrConstBindingList)BlockCatchoptFinallyopt
    • It is a Syntax Error if any element of the BoundNames of BindingList also occurs in the VarDeclaredNames of Block.

    2.3Static Semantics: ContainsDuplicateLabels

    With parameter labelSet.

    TryStatement:try(Expression)BlockCatchoptFinallyopt TryStatement:try(varVariableDeclarationList)BlockCatchoptFinallyopt TryStatement:try(LetOrConstBindingList)BlockCatchoptFinallyopt
    1. Let hasDuplicates be ContainsDuplicateLabels of Block with argument labelSet.
    2. If hasDuplicates is true, return true.
    3. If Catch is present, then
      1. Set hasDuplicates to ContainsDuplicateLabels of Catch with argument labelSet.
      2. If hasDuplicates is true, return true.
    4. If Finally is present, then
      1. Set hasDuplicates to ContainsDuplicateLabels of Finally with argument labelSet.
      2. If hasDuplicates is true, return true.
    5. Return false.

    2.6Static Semantics: VarDeclaredNames

    1. Let names be BoundNames of VariableDeclarationList.
    2. Append to names the elements of the VarDeclaredNames of Block.
    3. If Catch is present, then
      1. Append to names the elements of the VarDeclaredNames of Catch.
    4. If Finally is present, then
      1. Append to names the elements of the VarDeclaredNames of Finally.
    5. Return names.
    TryStatement:try(Expression)BlockCatchoptFinallyopt TryStatement:try(LetOrConstBindingList)BlockCatchoptFinallyopt
    1. Let names be VarDeclaredNames of Block.
    2. If Catch is present, then
      1. Append to names the elements of the VarDeclaredNames of Catch.
    3. If Finally is present, then
      1. Append to names the elements of the VarDeclaredNames of Finally.
    4. Return names.

    2.8Static Semantics: TryResources

    VariableDeclarationList:VariableDeclarationList,VariableDeclaration
    1. Let resources be TryResources of VariableDeclarationList.
    2. Append to resources the elements of the TryResources of VariableDeclaration.
    3. Return resources.
    VariableDeclarationList:VariableDeclaration
    1. Return « VariableDeclaration ».
    BindingList:BindingList,LexicalBinding
    1. Let resources be TryResources of BindingList.
    2. Append to resources the elements of the TryResources of LexicalBinding.
    3. Return resources.
    BindingList:LexicalBinding
    1. Return « LexicalBinding ».

    2.9Runtime Semantics: TryWithResourcesInstantiation (boundName, declarationKind, tryEnv)

    The abstract operation TryWithResourcesInstantiation with arguments boundName, declarationKind and tryEnv performs the following steps:

    1. Assert: tryEnv is a Lexical Environment.
    2. Assert: Type(boundName) is String or Undefined.
    3. Assert: declarationKind is "none", "var", "let" or "const".
    4. If declarationKind is "const" or "let", then
      1. Assert: Type(boundName) is String.
      2. Let envRec be tryEnv's EnvironmentRecord.
      3. Assert: envRec is a declarative EnvironmentRecord.
      4. If declarationKind is "const", then
        1. Perfom ! envRec.CreateImmutableBinding(boundName, true).
      5. Else,
        1. Perfom ! envRec.CreateMutableBinding(boundName, false).

    2.10Runtime Semantics: TryResourceEvaluation (resource, boundName, declarationKind)

    The abstract operation TryResourceEvaluation with arguments resource, boundName and declarationKind performs the following steps:

    1. Assert: declarationKind is "none", "var", "let", or "const".
    2. If declarationKind is "none", then
      1. Assert: resource is an Expression.
      2. Assert: boundName is undefined.
      3. Let exprRef be the result of evaluating resource.
      4. Return ? GetValue(exprRef).
    3. Else,
      1. Assert: resource is a VariableDeclaration or a LexicalBinding.
      2. Assert: Type(boundName) is String.
      3. Let decl be the result of evaluating resource.
      4. ReturnIfAbrupt(decl).
      5. Return ? ResolveBinding(boundName).

    2.11Runtime Semantics: Dispose ( obj, hint, completion )

    The abstract operation Dispose with arguments obj, hint and completion performs the following steps:

    1. Assert: hint is either async or sync.
    2. Assert: completion is a Completion Record.
    3. If obj is null or undefined, return completion.
    4. If hint is async, then
      1. Let method be GetMethod(obj, @@asyncDispose).
      2. If method.[[Type]] is normal and method.[[Value]] is undefined, then
        1. Set method to GetMethod(obj, @@dispose).
    5. Else,
      1. Set method to GetMethod(obj, @@dispose).
    6. If method.[[Type]] is normal, then
      1. Let result be Call(method.[[Value]], obj).
    7. Else,
      1. Let result be method.
    8. If result.[[Type]] is normal and hint is async, then
      1. Set result to Await(result.[[Value]]).
    9. If result.[[Type]] is throw, then
      1. If completion.[[Type]] is throw, then
        1. Let errors be CreateArrayFromList(« completion.[[Value]], result.[[Value]] »).
        2. Throw a new AggregateError exception with the errors in errors.
      2. Else,
        1. Return result.
    10. Else,
      1. Assert: result.[[Type]] is normal.
    11. Return completion.

    2.12Runtime Semantics: TryWithResourcesEvaluation (resources, boundNames, declarationKind, depth, Block)

    The abstract operation TryWithResourcesEvaluation with arguments resources, boundNames, declarationKind, depth, and Block performs the following steps:

    1. Assert: resources is a non-empty List.
    2. Assert: boundNames is a List.
    3. Assert: declarationKind is "none", "var", "let", or "const".
    4. Assert: Block is a Block.
    5. Let maxDepth be the number of elements in resources.
    6. Assert: 0 ≤ depth < maxDepth.
    7. Let resource be the element at index depth of resources.
    8. If declarationKind is "none", then
      1. Let boundName be undefined.
    9. Else,
      1. Assert: The number of elements in boundNames is equal to maxDepth.
      2. Let boundName be the element at index depth of boundNames.
    10. Let oldEnv be the running execution context's LexicalEnvironment.
    11. Let tryEnv be NewDeclarativeEnvironment(oldEnv).
    12. Perform TryWithResourcesInstantiation(boundName, declarationKind, tryEnv).
    13. Set the running execution context's LexicalEnvironment to tryEnv.
    14. Let exprCompletion be TryResourceEvaluation(resource, boundName).
    15. If exprCompletion.[[Type]] is not normal, then
      1. Set the running execution context's LexicalEnvironment to oldEnv.
      2. Return exprCompletion.
    16. If depth < maxDepth - 1, then
      1. Let result be TryWithResourcesEvaluation(resources, boundNames, declarationKind, depth + 1, Block).
    17. Else,
      1. Let result be the result of evaulating Block.
    18. If ! GetFunctionKind() is "async" or "async generator", then
      1. Let hint be async.
    19. Else,
      1. Let hint be sync.
    20. Let disposeResult be Dispose(expr, hint, result).
    21. Set the running execution context's LexicalEnvironment to oldEnv.
    22. Return disposeResult.

    2.13Runtime Semantics: Evaluation

    TryStatement:try(Expression)BlockCatchoptFinallyopt
    1. Let resources be « Expression ».
    2. Let B be TryWithResourcesEvaluation(resources, « », "none", 0, Block).
    3. If B.[[Type]] is throw and Catch is present, let C be CatchClauseEvaluation of Catch with argument B.[[Value]].
    4. Else, let C be B.
    5. If Finally is present, let F be the result of evaluating Finally.
    6. Else, let F be C.
    7. If F.[[Type]] is normal, set F to C.
    8. Return Completion(UpdateEmpty(F, undefined)).
    TryStatement:try(varVariableDeclarationList)BlockCatchoptFinallyopt
    1. Let resources be TryResources of VariableDeclarationList.
    2. Let boundNames be the BoundNames of VariableDeclarationList.
    3. Let B be TryWithResourcesEvaluation(resources, boundName, "var", 0, Block).
    4. If B.[[Type]] is throw and Catch is present, let C be CatchClauseEvaluation of Catch with argument B.[[Value]].
    5. Else, let C be B.
    6. If Finally is present, let F be the result of evaluating Finally.
    7. Else, let F be C.
    8. If F.[[Type]] is normal, set F to C.
    9. Return Completion(UpdateEmpty(F, undefined)).
    TryStatement:try(LetOrConstBindingList)BlockCatchoptFinallyopt
    1. Let resources be TryResources of BindingList.
    2. Let boundNames be the BoundNames of BindingList.
    3. If IsConstantDeclaration of LetOrConst is true, let declarationKind be "const".
    4. Else, let declarationKind be "let".
    5. Let B be TryWithResourcesEvaluation(resources, boundName, declarationKind, 0, Block).
    6. If B.[[Type]] is throw and Catch is present, let C be CatchClauseEvaluation of Catch with argument B.[[Value]].
    7. Else, let C be B.
    8. If Finally is present, let F be the result of evaluating Finally.
    9. Else, let F be C.
    10. If F.[[Type]] is normal, set F to C.
    11. Return Completion(UpdateEmpty(F, undefined)).

    3Fundamental Objects

    3.1Error Objects

    3.1.1AggregateError Objects

    An AggregateError Object is an extended NativeError object that encapsulates multiple errors.

    3.1.1.1The AggregateError Constructor

    The AggregateError constructor:

    • is the intrinsic object %AggregateError%
    • is the initial value of the AggregateError property of the global object.
    • creates and initializes a new AggregateError object when called as a function rather than as a constructor. Thus the function call AggregateError(...) is equivalent to the object creation expression new AggregateError(...) with the same arguments.
    • is designed to be subclassable. It may be used as the value of an extends clause of a class definition. Subclass constructors that intend to inherit the specified AggregateError behaviour must include a super call to the AggregateError constructor to create and initialize subclass instances with an [[ErrorData]] internal slot.

    3.1.1.1.1AggregateError ( errors, message )

    When the AggregateError function is called with arguments errors and message, the following steps are taken:

    1. If NewTarget is undefined, let newTarget be the active function object; else let newTarget be NewTarget.
    2. Let O be ? OrdinaryCreateFromConstructor(newTarget, "%AggregateErrorPrototype%", « [[ErrorData]] »).
    3. If errors is not undefined, then
      1. Let errorsArray be ? ArrayFrom( errors ).
    4. Else,
      1. Let errorsArray be ! ArrayCreate(0).
    5. Let errorsDesc be the PropertyDescriptor { [[Value]]: errorsArray, [[Writable]]: true, [[Enumerable]]: false, [[Configurable]]: true }.
    6. Perform ! DefinePropertyOrThrow(O, "errors", errorsDesc).
    7. If message is not undefined, then
      1. Let msg be ? ToString(message).
      2. Let msgDesc be the PropertyDescriptor { [[Value]]: msg, [[Writable]]: true, [[Enumerable]]: false, [[Configurable]]: true }.
      3. Perform ! DefinePropertyOrThrow(O, "message", msgDesc).
    8. Return O.

    3.1.1.2Properties of the AggregateError Constructor

    The AggregateError constructor:

    • has a [[Prototype]] internal slot whose value is the intrinsic object %Error%.
    • has the following properties:

    3.1.1.2.1AggregateError.prototype

    The intial value of AggregateError.prototype is the intrinsic object %AggregateErrorPrototype%.

    This property has the attributes { [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }.

    3.1.1.3Properties of the AggregateError Prototype Object

    The AggregateError prototype object:

    • is an ordinary object.
    • is not an Error instance and does not have an [[ErrorData]] internal slot.
    • has a [[Prototype]] internal slot whose value is the intrinsic object %ErrorPrototype%.

    3.1.1.3.1AggregateError.prototype.constructor

    The initial value of AggregateError.prototype.constructor is the intrinsic %AggregateError% object.

    3.1.1.3.2AggregateError.prototype.message

    The initial value of AggregateError.prototype.message is the empty String.

    3.1.1.3.3AggregateError.prototype.name

    The initial value of AggregateError.prototype.name is "AggregateError".

    3.1.1.3.4AggregateError.prototype.errors

    The initial value of AggregateError.prototype.errors is undefined.

    3.1.1.4Properties of AggregateError Instances

    AggregateError instances are ordinary objects that inherit properties from the AggregateError prototype object and have an [[ErrorData]] internal slot whose value is undefined. The only specified uses of [[ErrorData]] is to identify Error, AggregateError, and NativeError instances as Error objects within Object.prototype.toString.

    4Indexed Collections

    4.1Array Objects

    4.1.1Properties of the Array Constructor

    4.1.1.1Array.from ( items [ , mapfn [ , thisArg ] ] )

    When the from method is called with argument items and optional arguments mapfn and thisArg, the following steps are taken:

    1. Let C be the this value.
    2. If mapfn is undefined, let mapping be false.
    3. Else,
      1. If IsCallable(mapfn) is false, throw a TypeError exception.
      2. If thisArg is present, let T be thisArg; else let T be undefined.
      3. Let mapping be true.
    4. Let usingIterator be ? GetMethod(items, @@iterator).
    5. If usingIterator is not undefined, then
      1. If IsConstructor(C) is true, then
        1. Let A be ? Construct(C).
      2. Else,
        1. Let A be ! ArrayCreate(0).
      3. Let iteratorRecord be ? GetIterator(items, sync, usingIterator).
      4. Let k be 0.
      5. Repeat,
        1. If k ≥ 253 - 1, then
          1. Let error be ThrowCompletion(a newly created TypeError object).
          2. Return ? IteratorClose(iteratorRecord, error).
        2. Let Pk be ! ToString(k).
        3. Let next be ? IteratorStep(iteratorRecord).
        4. If next is false, then
          1. Perform ? Set(A, "length", k, true).
          2. Return A.
        5. Let nextValue be ? IteratorValue(next).
        6. If mapping is true, then
          1. Let mappedValue be Call(mapfn, T, « nextValue, k »).
          2. If mappedValue is an abrupt completion, return ? IteratorClose(iteratorRecord, mappedValue).
          3. Set mappedValue to mappedValue.[[Value]].
        7. Else, let mappedValue be nextValue.
        8. Let defineStatus be CreateDataPropertyOrThrow(A, Pk, mappedValue).
        9. If defineStatus is an abrupt completion, return ? IteratorClose(iteratorRecord, defineStatus).
        10. Increase k by 1.
    6. NOTE: items is not an Iterable so assume it is an array-like object.
    7. Let arrayLike be ! ToObject(items).
    8. Let len be ? ToLength(? Get(arrayLike, "length")).
    9. If IsConstructor(C) is true, then
      1. Let A be ? Construct(C, « len »).
    10. Else,
      1. Let A be ? ArrayCreate(len).
    11. Let k be 0.
    12. Repeat, while k < len
      1. Let Pk be ! ToString(k).
      2. Let kValue be ? Get(arrayLike, Pk).
      3. If mapping is true, then
        1. Let mappedValue be ? Call(mapfn, T, « kValue, k »).
      4. Else, let mappedValue be kValue.
      5. Perform ? CreateDataPropertyOrThrow(A, Pk, mappedValue).
      6. Increase k by 1.
    13. Perform ? Set(A, "length", len, true).
    14. Return A.
    15. Return ? ArrayFrom(items, C, mapfn, thisArg).

    4.1.1.1.1ArrayFrom ( items [ , C [ , mapfn [ , thisArg ] ] ] )

    The abstract operation ArrayFrom with arguments items, C, mapfn, and thisArg performs the following steps:

    1. If mapfn is undefined, let mapping be false.
    2. Else,
      1. If IsCallable(mapfn) is false, throw a TypeError exception.
      2. If thisArg is present, let T be thisArg; else let T be undefined.
      3. Let mapping be true.
    3. Let usingIterator be ? GetMethod(items, @@iterator).
    4. If usingIterator is not undefined, then
      1. If IsConstructor(C) is true, then
        1. Let A be ? Construct(C).
      2. Else,
        1. Let A be ! ArrayCreate(0).
      3. Let iteratorRecord be ? GetIterator(items, sync, usingIterator).
      4. Let k be 0.
      5. Repeat,
        1. If k ≥ 253 - 1, then
          1. Let error be ThrowCompletion(a newly created TypeError object).
          2. Return ? IteratorClose(iteratorRecord, error).
        2. Let Pk be ! ToString(k).
        3. Let next be ? IteratorStep(iteratorRecord).
        4. If next is false, then
          1. Perform ? Set(A, "length", k, true).
          2. Return A.
        5. Let nextValue be ? IteratorValue(next).
        6. If mapping is true, then
          1. Let mappedValue be Call(mapfn, T, « nextValue, k »).
          2. If mappedValue is an abrupt completion, return ? IteratorClose(iteratorRecord, mappedValue).
          3. Set mappedValue to mappedValue.[[Value]].
        7. Else, let mappedValue be nextValue.
        8. Let defineStatus be CreateDataPropertyOrThrow(A, Pk, mappedValue).
        9. If defineStatus is an abrupt completion, return ? IteratorClose(iteratorRecord, defineStatus).
        10. Increase k by 1.
    5. NOTE: items is not an Iterable so assume it is an array-like object.
    6. Let arrayLike be ! ToObject(items).
    7. Let len be ? ToLength(? Get(arrayLike, "length")).
    8. If IsConstructor(C) is true, then
      1. Let A be ? Construct(C, « len »).
    9. Else,
      1. Let A be ? ArrayCreate(len).
    10. Let k be 0.
    11. Repeat, while k < len
      1. Let Pk be ! ToString(k).
      2. Let kValue be ? Get(arrayLike, Pk).
      3. If mapping is true, then
        1. Let mappedValue be ? Call(mapfn, T, « kValue, k »).
      4. Else, let mappedValue be kValue.
      5. Perform ? CreateDataPropertyOrThrow(A, Pk, mappedValue).
      6. Increase k by 1.
    12. Perform ? Set(A, "length", len, true).
    13. Return A.
